# DecodeLabs-Internship
This project wasn't just about writing code; it was a deep dive into applying "Industrial Engineering" principles to software development.
This To-Do List isn't your average basic script. It's a console-based application that features:
✅ Persistent Storage: Tasks are saved to a tasks.json file, so nothing gets lost when the app closes. We moved from "Volatile Memory" to "Persistent Memory" to ensure data integrity.
➕ Add Tasks: Easily add new tasks to your list.
👀 View Tasks: See all your tasks, clearly numbered and with their completion status.
🗑️ Delete Tasks: Remove tasks you no longer need.
🎯 Mark as Completed: Update the status of tasks without deleting them, showcasing "State Mutation" within our "Data Objects."
⚙️ Menu-Driven Interface: An intuitive "Control Loop" guides user interaction, making it user-friendly.
🔄 Data Migration & Schema Evolution: The application gracefully handles older data formats, automatically updating them to a more structured dictionary-based schema, ensuring future extensibility.
Through this project, I've gained hands-on experience with fundamental concepts like the IPO Framework (Input, Process, Output), Control Loops, Conditional Logic, the Iterator Protocol, and managing data with "Volatile Containers" (lists) and "Data Objects" (dictionaries). We even tackled the nuances of "Human-Centric" vs. "Zero-based Indexing" for a better user experience!
